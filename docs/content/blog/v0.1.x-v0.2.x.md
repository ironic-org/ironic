# v0.1.x — v0.2.x: The pre-release journey

Before the stable v0.3.x series, Ironic went through 19 pre-release iterations. Here's how the framework evolved.

---

## v0.1.4 — First public release (Jul 13, 2026)

The initial release shipped with the complete architectural foundation:

### Core
- **9-crate workspace**: DI container, HTTP contracts, platform boundary, Axum adapter, testing harness, macros, CLI, and the ironic facade crate
- **Module system** (RFC 0001) — validated provider/controller/import/export graphs
- **Dependency injection** (RFC 0002) — `#[derive(Injectable)]` with singleton and transient scopes
- **Controller routing** (RFC 0003) — `#[controller]`, `#[get]`, `#[post]`, `#[put]`, `#[delete]`
- **Request lifecycle** (RFC 0004) — Middleware → Guards → Interceptors → Extraction → Pipes → Handler
- **Platform adapter** (RFC 0005) — Axum adapter with `configure_router()` escape hatch

### Features (all opt-in via Cargo features)
- **Security**: CORS, rate limiting, CSRF, security headers
- **Validation**: `ParseIntPipe`, `ParseFloatPipe`, `ParseBoolPipe`, `ValidationPipe` with `garde`
- **API versioning**: URI prefix, request header, and media type strategies
- **Exception filters**: Typed error handling with route metadata access and scope precedence
- **Database backends**: SQLx, SeaORM, Diesel, MongoDB, Redis
- **Authentication**: Argon2, JWT, OAuth2, sessions
- **Services**: Caching, scheduling, events, WebSocket gateways, realtime
- **Distributed**: Queues, microservices, CQRS, sagas, gRPC, GraphQL
- **OpenAPI 3.1**: Route discovery, Swagger UI
- **Observability**: Health endpoints, request correlation spans

---

## v0.2.x — Refinement (Jul 14–15, 2026)

The v0.2.x series focused on polishing the developer experience:

### v0.2.6–v0.2.9
- **Ready-resource generators** — `ironic generate rr auth|auth-basic|auth-jwt|auth-oauth|file-upload|email`
- **Authentication modules** — Password hashing, JWT tokens, OAuth2 social login, session management
- **File upload module** — Local, S3, R2, Azure Blob, GCS backends
- **Email module** — SMTP, SES, SendGrid, Mailgun, and dev-log backends
- **Workspace command** — `ironic workspace` prints project info, modules, and version
- **Dotenvy support** — `.env` loading with `SERVER_HOST` and `SERVER_PORT` config

### v0.2.5–v0.2.8
- **Project scaffold** — Generated projects include example CRUD module, CI workflow, Dockerfile
- **Changelog generation** — Automated release notes from commit history
- **Poisoned mutex handling** — Resilience across metrics, rate limiting, and security modules
- **CORS defaults** — Deny-by-default with explicit origin allowlisting
- **CSRF improvements** — Configurable cookie and header name validation

---

## What changed in v0.3.0

The jump to v0.3.0 marked the first stable release. All pre-release APIs were solidified, the module graph compiler was hardened, and the CLI reached feature completeness with `ironic generate` covering 12 resource types.

The framework hasn't fundamentally changed since v0.1.4 — the architecture was right from the start. v0.2.x was about filling gaps, and v0.3.x is about polish, documentation, and production readiness.
