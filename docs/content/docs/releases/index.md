## Versioning policy

- **Major** (0.x → 1.0): Breaking API changes, significant design shifts
- **Minor** (0.3 → 0.4): New features, non-breaking additions
- **Patch** (0.3.7 → 0.3.8): Bug fixes, docs, small improvements
